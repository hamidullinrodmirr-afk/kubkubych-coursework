# Техническое задание
## Веб-приложение "PetCare" - Ветеринарная клиника

---

## 1. Общее описание проекта

**PetCare** - полнофункциональное веб-приложение для ветеринарной клиники, предоставляющее клиентам возможность просматривать каталог услуг и врачей, записываться на приём онлайн, а администрации клиники - управлять расписанием, персоналом и аналитикой.

### Стек технологий
- **Backend**: Python 3.10+, Django 4.x+, Django REST Framework
- **СУБД**: PostgreSQL
- **Асинхронные задачи**: Celery + Redis
- **Почта**: Mailhog (тестирование email)
- **Мониторинг**: Sentry (отслеживание ошибок), Django Silk (профилирование)
- **Аутентификация**: JWT + OAuth2 (ВКонтакте, Google)
- **Тестирование API**: Postman
- **Frontend**: HTML/CSS/JS (по макетам из Figma)

## 2. Роли пользователей

### 2.1. Клиент (владелец питомца)

Клиент - зарегистрированный пользователь, обращающийся в клинику за ветеринарными услугами.

| # | Возможность | Описание |
|---|-------------|----------|
| 1 | **Просмотр каталога услуг и врачей** | Просмотр списка ветеринаров с фильтрацией по специальности (кардиология, дерматология, неврология и др.), просмотр услуг с ценами, поиск по имени врача |
| 2 | **Запись на приём** | Выбор врача, услуги, даты и времени приёма. Заполнение формы с указанием ФИО, телефона, email, комментария. Просмотр доступных слотов в расписании врача |
| 3 | **Управление питомцами** | Добавление, редактирование и удаление профилей своих питомцев (кличка, вид животного, порода, возраст, особенности здоровья) |
| 4 | **Личный кабинет** | Просмотр и редактирование личных данных (ФИО, телефон, email). Просмотр истории приёмов и статусов текущих записей ("ожидание", "подтверждён", "завершён", "отменён") |
| 5 | **Отзывы и оценки** | Написание отзывов о посещённых врачах и выставление оценок (1-5 звёзд). Редактирование и удаление собственных отзывов |
| 6 | **Уведомления** | Получение email-уведомлений о подтверждении записи, напоминании о приёме за 24 часа, изменении статуса записи |

### 2.2. Ветеринар (врач)

Ветеринар - сотрудник клиники, оказывающий ветеринарные услуги.

| # | Возможность | Описание |
|---|-------------|----------|
| 1 | **Управление расписанием** | Просмотр своего расписания на день/неделю/месяц. Просмотр записей пациентов с деталями (питомец, услуга, владелец) |
| 2 | **Управление профилем** | Редактирование профессиональной информации: образование, стаж, специализации, описание. Загрузка фотографии |
| 3 | **Ведение медицинских карт** | Создание и редактирование записей в медкартах питомцев: диагнозы, назначения, рекомендации, результаты осмотров |
| 4 | **Управление статусами приёмов** | Изменение статуса записи: "ожидание" -> "подтверждён" -> "в процессе" -> "завершён". Отмена приёма с указанием причины |
| 5 | **Просмотр отзывов** | Просмотр отзывов, оставленных клиентами о своей работе. Просмотр своего среднего рейтинга |

### 2.3. Администратор

Администратор - сотрудник клиники с полными правами управления системой.

| # | Возможность | Описание |
|---|-------------|----------|
| 1 | **Управление врачами** | Добавление, редактирование и удаление профилей ветеринаров. Назначение специализаций. Настройка расписания работы врачей (дни недели, часы приёма) |
| 2 | **Управление услугами** | Создание, редактирование и удаление услуг. Установка и изменение цен. Привязка услуг к специализациям. Управление категориями услуг |
| 3 | **Управление записями** | Просмотр всех записей в системе. Отмена, перенос записей. Создание записей от имени клиента (по звонку). Разрешение конфликтов расписания |
| 4 | **Модерация отзывов** | Просмотр, одобрение, редактирование и удаление отзывов клиентов. Фильтрация отзывов по рейтингу и дате |
| 5 | **Управление пользователями** | Просмотр списка всех пользователей. Блокировка/разблокировка аккаунтов. Изменение ролей пользователей |
| 6 | **Аналитика и отчёты** | Просмотр статистики: популярность услуг, загрузка врачей, количество записей по периодам, средние рейтинги врачей, доход клиники |
| 7 | **Настройки клиники** | Управление общей информацией: название, адрес, контакты, режим работы. Управление контентом сайта |

## 4. Бизнес-логика и валидация

### 4.1. Валидация доступности временного слота
**Описание**: При создании записи на приём проверяется, что выбранный временной слот свободен у указанного врача. Не допускается создание двух записей на одно и то же время у одного врача.

**Правила**:
- Проверка отсутствия пересечений с существующими записями (status != cancelled)
- Проверка, что выбранное время входит в расписание врача (Schedule)
- Проверка, что дата приёма не в прошлом
- Проверка, что дата приёма не более чем на 30 дней вперёд

### 4.2. Валидация соответствия врача и услуги
**Описание**: При записи на приём проверяется, что выбранный врач может оказать выбранную услугу, т.е. его специализация соответствует специализации услуги.

**Правила**:
- Специализация услуги (service.specialty) должна входить в список специализаций врача (doctor.specialties)
- Врач должен быть активен (is_available=True)

### 4.3. Валидация отзывов
**Описание**: Оставлять отзыв может только клиент, который действительно посещал данного врача и приём имеет статус "завершён".

**Правила**:
- У автора отзыва должен существовать хотя бы один завершённый приём (status=completed) у данного врача
- Один приём - один отзыв (нельзя оставить два отзыва на один приём)
- Оценка строго от 1 до 5
- Текст отзыва не менее 10 символов

### 4.4. Валидация прав доступа к записям
**Описание**: Клиент может просматривать и управлять только своими записями. Ветеринар видит только записи, назначенные на него. Администратор имеет доступ ко всем записям.

**Правила**:
- Клиент: доступ только к записям, где client=текущий_пользователь
- Ветеринар: доступ только к записям, где doctor.user=текущий_пользователь
- Администратор: доступ ко всем записям
- При попытке доступа к чужой записи - HTTP 403 Forbidden

### 4.5. Валидация данных питомца
**Описание**: При создании/редактировании профиля питомца проверяется корректность введённых данных.

**Правила**:
- Возраст питомца: от 0 до 360 месяцев (30 лет)
- Вес: от 0.01 до 200 кг
- Кличка: от 1 до 50 символов, только буквы и пробелы
- Вид животного: только из допустимого списка (cat, dog, bird, rodent, reptile, other)
- Питомец принадлежит только текущему авторизованному клиенту

### 4.6. Валидация расписания врача
**Описание**: При настройке расписания проверяется корректность временных интервалов.

**Правила**:
- Время начала приёма должно быть раньше времени окончания
- Не допускается пересечение временных интервалов для одного врача в один день
- Длительность слота: от 15 до 120 минут

## 5. Оптимизация запросов (select_related / prefetch_related)

### 5.1. Получение списка врачей со специализациями
```python
Doctor.objects.prefetch_related('specialties').select_related('user')
```
Используется на страницах: главная, каталог врачей.

### 5.2. Получение записей с информацией о клиенте и враче
```python
Appointment.objects.select_related('client', 'doctor__user', 'pet', 'service')
```
Используется: личный кабинет, расписание врача, админ-панель.

### 5.3. Получение отзывов с информацией о враче и авторе
```python
Review.objects.select_related('author', 'doctor__user', 'appointment')
```
Используется: страница врача, модерация отзывов.

### 5.4. Получение медицинских карт с полной информацией
```python
MedicalRecord.objects.select_related('appointment__client', 'pet', 'doctor__user')
```
Используется: история посещений питомца.

## 6. Сериализаторы

### 6.1. SerializerMethodField
Используется для вычисляемых полей:

```python
class DoctorListSerializer(serializers.ModelSerializer):
    full_name = serializers.SerializerMethodField()
    avg_rating = serializers.SerializerMethodField()
    reviews_count = serializers.SerializerMethodField()

    def get_full_name(self, obj):
        """Полное имя врача."""
        return f"{obj.user.last_name} {obj.user.first_name}"

    def get_avg_rating(self, obj):
        """Средний рейтинг врача."""
        return obj.reviews.aggregate(avg=Avg('rating'))['avg'] or 0

    def get_reviews_count(self, obj):
        """Количество отзывов."""
        return obj.reviews.filter(is_approved=True).count()
```

## 7. Аннотации

Аннотации используются для эффективного вычисления агрегированных данных на уровне БД.

### 7.1. Средний рейтинг врача
```python
from django.db.models import Avg, Count

Doctor.objects.annotate(
    avg_rating=Avg('reviews__rating'),
    reviews_count=Count('reviews', filter=Q(reviews__is_approved=True))
)
```

### 7.2. Количество приёмов врача за период
```python
Doctor.objects.annotate(
    appointments_count=Count(
        'appointments',
        filter=Q(appointments__date__gte=start_date, appointments__date__lte=end_date)
    )
)
```

Service.objects.annotate(

class DoctorFilter(django_filters.FilterSet):

class ServiceFilter(django_filters.FilterSet):

class AppointmentFilter(django_filters.FilterSet):

## 10. Периодические задачи (Celery)

| Задача | Периодичность | Описание |
|--------|---------------|----------|
| `send_appointment_reminder` | Каждый час | Отправка email-напоминания клиентам за 24 часа до приёма |
| `cancel_unconfirmed_appointments` | Каждые 6 часов | Автоотмена записей, не подтверждённых врачом в течение 48 часов |
| `generate_daily_report` | Ежедневно в 23:00 | Генерация отчёта по приёмам за день для администратора |
| `cleanup_expired_slots` | Ежедневно в 00:00 | Очистка просроченных слотов расписания |
